1. Day04에서 배우는 Java 문법

Day04는 실행 중에 클래스 정보를 다루는 리플렉션(reflection)과 코드를 더 바꾸기 쉽게 만드는 느슨한 결합을 배웁니다. Day01의 인터페이스 개념이 Day04에서 설정 파일, 리플렉션, 의존성 주입과 연결됩니다.

개념쉬운 뜻대표 코드
Class 객체클래스 정보를 담는 객체Class.forName("패키지.클래스")
Reflection실행 중 클래스 정보를 보고 사용하는 기술getDeclaredMethods()
Method invoke메서드를 이름으로 찾아 실행method.invoke(obj)
Properties외부 설정 파일을 key-value로 읽음p.getProperty("animal")
느슨한 결합구체 클래스가 아니라 인터페이스에 의존Calculator calculator;
핵심: Day04의 목표는 “코드에 클래스 이름을 직접 박아 넣지 않고도 실행 시점에 필요한 객체를 선택하는 구조”를 이해하는 것입니다.

2. 리플렉션이란?

리플렉션은 실행 중에 클래스 정보를 거울처럼 들여다보는 기술입니다. 클래스 이름, 필드, 메서드, 생성자 정보를 읽을 수 있고, 객체를 만들거나 메서드를 호출할 수도 있습니다. 

Class clazz = Class.forName("day4.reflect.Member");

위 코드는 문자열로 적힌 클래스 이름을 이용해서 Member 클래스 정보를 가져옵니다. 여기서 중요한 점은 컴파일할 때가 아니라 실행 중에 클래스 정보를 읽는다는 것입니다.

4. 예제 클래스 Member

public class Member {
    String id;
    String pw;

    public Member() {
        this("기본아이디", "기본비번");
    }

    public Member(String id, String pw) {
        this.id = id;
        this.pw = pw;
    }

    public void printInfo() {
        System.out.println("id=" + id);
        System.out.println("pw=" + pw);
    }

    public void printSquare(int su) {
        System.out.println(su * su);
    }
}

이 클래스는 리플렉션 실습용입니다. 필드 정보, 메서드 정보, 객체 생성, 메서드 호출을 모두 연습할 수 있습니다.

5. 필드와 메서드 정보 얻기

Class clazz = Class.forName("day4.reflect.Member");

Field[] fields = clazz.getDeclaredFields();
for (Field field : fields) {
    System.out.println(field);
}

Method[] methods = clazz.getDeclaredMethods();
for (Method method : methods) {
    System.out.println(method.getName());
}
메서드뜻
getDeclaredFields()클래스에 선언된 필드 목록을 가져옴
getDeclaredMethods()클래스에 선언된 메서드 목록을 가져옴
getDeclaredMethod("이름", 타입...)특정 메서드 하나를 가져옴

6. 리플렉션으로 객체 만들기

Class clazz = Member.class;
Member member = (Member) clazz.newInstance();
member.printSquare(3);

newInstance()는 기본 생성자를 사용해서 객체를 만듭니다. 최근 Java에서는 getDeclaredConstructor().newInstance() 사용을 더 권장하지만, 수업 예제에서는 리플렉션 개념 이해를 위해 newInstance()를 사용합니다. 

Class clazz = Class.forName("day4.reflect.Member");
Object obj = clazz.getDeclaredConstructor().newInstance();
일반 객체 생성: new Member()
리플렉션 객체 생성: 클래스 정보를 얻은 뒤 실행 중 객체 생성

7. 리플렉션으로 메서드 호출하기

Member m = new Member();
Class clazz = Class.forName("day4.reflect.Member");

Method method = clazz.getDeclaredMethod("printInfo");
method.invoke(m);

Method method2 = clazz.getDeclaredMethod("printSquare", int.class);
method2.invoke(m, 3);

invoke는 메서드를 실행합니다. 첫 번째 인수에는 메서드를 실행할 객체가 들어가고, 두 번째 이후 인수에는 메서드에 전달할 값이 들어갑니다.

주의: 리플렉션은 강력하지만 오타나 타입 오류가 실행 중에 터질 수 있습니다. 그래서 일반 코드보다 더 신중하게 써야 합니다.

8. 변경에 유리한 코드

변경에 유리한 코드는 특정 클래스에 강하게 묶이지 않는 코드입니다. 예를 들어 new Dog()를 코드 곳곳에 직접 쓰면 Cat으로 바꿀 때 여러 곳을 수정해야 합니다. 대신 설정 파일과 리플렉션을 사용하면 외부 파일만 바꿔도 구현체를 바꿀 수 있습니다. 

public class Animal {
    public void bark() {
        System.out.println("동물이 짖는다");
    }
}

public class Dog extends Animal {
    @Override
    public void bark() {
        System.out.println("멍멍");
    }
}

public class Cat extends Animal {
    @Override
    public void bark() {
        System.out.println("야옹");
    }
}
// config.txt
animal=day4.변경에유리한코드.Dog
Properties p = new Properties();
p.load(new FileReader("src/day4/변경에유리한코드/config.txt"));

String className = p.getProperty("animal");
Class clazz = Class.forName(className);
Animal animal = (Animal) clazz.newInstance();

animal.bark();

이제 Dog 대신 Cat을 쓰고 싶으면 Java 코드를 고치는 대신 설정 파일의 클래스명만 바꾸면 됩니다.

9. 느슨한 결합: 계산기 예제

계산기 예제는 Day04의 핵심 구조를 잘 보여줍니다. 프로그램이 특정 계산기 클래스에 직접 의존하지 않고, Calculator 인터페이스에 의존합니다. 

public interface Calculator {
    int add(int su1, int su2);
}
public class C implements Calculator {
    @Override
    public int add(int su1, int su2) {
        System.out.println("C add");
        return su1 + su2;
    }
}

public class D implements Calculator {
    @Override
    public int add(int su1, int su2) {
        System.out.println("D add");
        return su1 + su2;
    }
}
public class Program {
    Calculator calculator;

    public void setCalculator(Calculator calculator) {
        this.calculator = calculator;
    }

    public void run() {
        int total = calculator.add(10, 20);
        System.out.println(total);
    }
}

프로그램은 C인지 D인지 몰라도 됩니다. Calculator 인터페이스의 add만 호출하면 됩니다.

구조문제점/장점
new C()를 프로그램 안에 직접 작성C에서 D로 바꾸려면 Java 코드 수정 필요
Calculator 인터페이스에 의존C, D, SMCalculator 등 구현체 교체가 쉬움
설정 파일 + 리플렉션코드 수정 없이 외부 파일로 구현체 선택 가능

Properties properties = new Properties();
