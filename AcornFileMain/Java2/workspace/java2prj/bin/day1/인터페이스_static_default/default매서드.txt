한 것이 default 메서드의 가장 큰 존재 "



default  매서드 사용 이유 

0) 가장 근본적인 이유
   기존 인터페이스를 깨뜨리지 않고 기능 확장 가능
   새로운 메서드를 추가해도 기존 구현 클래스가 컴파일 에러 없이 그대로 동작
   Java 8에서 default method가 도입된 가장 큰 이유


1) 공통 로직 제공
   구현 클래스마다 반복 구현하지 않도록 인터페이스에서 기본 구현 제공


2) 기본 기능 제공 + 필요 시 오버라이드 가능
   구현 클래스는 그대로 사용하거나 원하는 경우 재정의 가능


3) 템플릿 메서드 패턴 비슷한 구조 가능
   인터페이스에서도 작업 흐름을 정의할 수 있음
