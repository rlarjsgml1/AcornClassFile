01

라이브러리를 만들 때 인터페이스 활용하기

인터페이스는 클래스가 반드시 지켜야 하는 사용 설명서 또는 약속 목록입니다. 특히 라이브러리를 만들 때는 사용자가 어떤 클래스를 만들지 미리 알 수 없으므로, “이 기능을 가진 객체라면 이런 메서드를 사용할 수 있어야 한다”라는 규칙을 인터페이스로 정해 둡니다.

비유: 리모컨 규격

TV 회사가 달라도 리모컨에 전원 버튼이 있으면 사용자는 전원을 켤 수 있습니다. Java의 인터페이스도 비슷합니다. 라이브러리는 “전원 버튼 규칙”만 알고 있으면 되고, 실제 TV가 어떤 회사 제품인지는 몰라도 됩니다.

interface Printable {
    void print();
}

class Report implements Printable {
    public void print() {
        System.out.println("보고서를 출력합니다.");
    }
}

class PrintLibrary {
    static void run(Printable target) {
        target.print();
    }
}

초보자가 자주 헷갈리는 점

interface 안의 추상 메서드는 기본적으로 public abstract입니다. 그래서 구현 클래스에서는 접근 범위를 줄이면 안 됩니다. 보통 public을 붙여 구현합니다.

02

인터페이스를 왜 쓰는가?

인터페이스를 사용하면 코드를 “구체적인 클래스”보다 “공통 기능” 중심으로 작성할 수 있습니다. 이렇게 하면 새로운 클래스가 추가되어도 기존 코드를 많이 고치지 않아도 됩니다.

interface Speaker {
    void speak();
}

class KoreanTeacher implements Speaker {
    public void speak() {
        System.out.println("안녕하세요.");
    }
}

class EnglishTeacher implements Speaker {
    public void speak() {
        System.out.println("Hello.");
    }
}

class Main {
    public static void main(String[] args) {
        Speaker teacher = new KoreanTeacher();
        teacher.speak();
    }
}

실행 흐름

  1. Speaker teacher 변수는 speak()를 사용할 수 있는 객체를 담겠다고 선언합니다.
  2. new KoreanTeacher()가 실제 객체를 만듭니다.
  3. teacher.speak()를 호출하면 실제 객체인 KoreanTeacher의 메서드가 실행됩니다.

03

ArrayList<Acorn> 준비하기

정렬을 배우기 위해 도토리 정보를 담는 Acorn 클래스를 사용합니다. ArrayList<Acorn>은 Acorn 객체만 담는 목록입니다.

import java.util.ArrayList;

class Acorn {
    String name;
    int weight;
    int score;

    Acorn(String name, int weight, int score) {
        this.name = name;
        this.weight = weight;
        this.score = score;
    }

    public String toString() {
        return name + "(" + weight + "g, " + score + "점)";
    }
}

class Main {
    public static void main(String[] args) {
        ArrayList<Acorn> acorns = new ArrayList<>();
        acorns.add(new Acorn("가람", 3, 80));
        acorns.add(new Acorn("나래", 8, 95));
        acorns.add(new Acorn("다온", 5, 70));

        System.out.println(acorns);
    }
}
표현 뜻
ArrayList 크기가 자동으로 늘어나는 목록
<Acorn> 이 목록에는 Acorn만 넣겠다는 타입 약속
new ArrayList<>() 오른쪽의 빈 꺾쇠는 왼쪽 타입을 보고 Java가 추측한다는 뜻

04

Comparable로 기본 정렬 만들기

Comparable<Acorn>은 “Acorn 객체끼리 비교할 때 기본 기준은 이것이다”라고 클래스 안에 적는 방식입니다.

import java.util.ArrayList;
import java.util.Collections;

class Acorn implements Comparable<Acorn> {
    String name;
    int weight;
    int score;

    Acorn(String name, int weight, int score) {
        this.name = name;
        this.weight = weight;
        this.score = score;
    }

    public int compareTo(Acorn other) {
        return this.score - other.score;
    }

    public String toString() {
        return name + "(" + score + "점)";
    }
}

class Main {
    public static void main(String[] args) {
        ArrayList<Acorn> acorns = new ArrayList<>();
        acorns.add(new Acorn("나래", 8, 95));
        acorns.add(new Acorn("가람", 3, 80));
        acorns.add(new Acorn("다온", 5, 70));

        Collections.sort(acorns);
        System.out.println(acorns);
    }
}

정렬 실행 흐름

  1. Collections.sort(acorns)가 목록을 정렬하려고 합니다.
  2. Acorn이 Comparable<Acorn>을 구현했는지 확인합니다.
  3. 각 도토리의 compareTo를 호출해 서로 비교합니다.
  4. 결과가 음수이면 앞, 양수이면 뒤, 0이면 같은 순서로 판단합니다.

Common Confusion

this.score - other.score는 점수 오름차순입니다. 반대로 높은 점수가 먼저 오게 하려면 other.score - this.score를 사용합니다.

05

Comparator로 상황별 정렬 만들기

Comparator<Acorn>는 클래스 밖에서 정렬 기준을 따로 만드는 방식입니다. “오늘은 이름순, 내일은 점수순”처럼 상황마다 기준이 바뀔 때 좋습니다.

import java.util.ArrayList;
import java.util.Collections;
import java.util.Comparator;

class Acorn {
    String name;
    int weight;
    int score;

    Acorn(String name, int weight, int score) {
        this.name = name;
        this.weight = weight;
        this.score = score;
    }

    public String toString() {
        return name + "(" + score + "점)";
    }
}

class NameComparator implements Comparator<Acorn> {
    public int compare(Acorn first, Acorn second) {
        return first.name.compareTo(second.name);
    }
}

class ScoreComparator implements Comparator<Acorn> {
    public int compare(Acorn first, Acorn second) {
        return second.score - first.score;
    }
}

class Main {
    public static void main(String[] args) {
        ArrayList<Acorn> acorns = new ArrayList<>();
        acorns.add(new Acorn("나래", 8, 95));
        acorns.add(new Acorn("가람", 3, 80));
        acorns.add(new Acorn("다온", 5, 70));

        Collections.sort(acorns, new NameComparator());
        System.out.println("이름순: " + acorns);

        Collections.sort(acorns, new ScoreComparator());
        System.out.println("점수순: " + acorns);
    }
}
구분 Comparable Comparator
위치 정렬될 클래스 안 별도 클래스 또는 익명 객체
메서드 compareTo compare
용도 대표 기본 정렬 1개 상황별 정렬 여러 개

06

interface default/static 메서드

인터페이스도 Java 8부터 구현 내용을 가진 메서드를 가질 수 있습니다. default 메서드는 구현 클래스 객체로 호출하고, static 메서드는 인터페이스 이름으로 호출합니다.

interface StudyTool {
    void study();

    default void rest() {
        System.out.println("10분 쉬기");
    }

    static void guide() {
        System.out.println("짧게 반복해서 공부하세요.");
    }
}

class JavaBook implements StudyTool {
    public void study() {
        System.out.println("Java 예제를 따라 칩니다.");
    }
}

class Main {
    public static void main(String[] args) {
        JavaBook book = new JavaBook();
        book.study();
        book.rest();
        StudyTool.guide();
    }
}

Common Confusion

static 메서드는 객체가 아니라 인터페이스 이름으로 호출합니다. book.guide()가 아니라 StudyTool.guide()입니다.

07

제네릭 클래스

제네릭 클래스는 타입을 미리 고정하지 않고, 사용할 때 타입을 정하는 클래스입니다. 같은 보관함 코드를 String, Integer, Acorn 등 여러 타입에 안전하게 재사용할 수 있습니다.

비유: 이름표 붙이는 상자

빈 상자에 “도토리 전용”이라고 붙이면 도토리만 넣습니다. “문자열 전용”이라고 붙이면 문자열만 넣습니다. 제네릭의 <T>는 상자에 붙일 이름표 자리입니다.

class Box<T> {
    private T item;

    public void setItem(T item) {
        this.item = item;
    }

    public T getItem() {
        return item;
    }
}

class Main {
    public static void main(String[] args) {
        Box<String> textBox = new Box<>();
        textBox.setItem("Hello");

        Box<Acorn> acornBox = new Box<>();
        acornBox.setItem(new Acorn("작은도토리", 3, 100));
    }
}

T는 특별한 예약어가 아니라 관습적으로 많이 쓰는 타입 이름입니다. E, K, V도 자주 사용합니다.

08

제한된 제네릭

제네릭 타입에 아무 타입이나 들어오면 사용할 수 있는 메서드가 너무 적습니다. extends로 “이 타입 또는 그 자식만 가능”이라고 제한하면 더 안전하게 기능을 사용할 수 있습니다.

class NumberBox<T extends Number> {
    private T value;

    void setValue(T value) {
        this.value = value;
    }

    double doubleValue() {
        return value.doubleValue();
    }
}

class Main {
    public static void main(String[] args) {
        NumberBox<Integer> intBox = new NumberBox<>();
        intBox.setValue(10);

        NumberBox<Double> doubleBox = new NumberBox<>();
        doubleBox.setValue(3.14);
    }
}

왜 제한이 필요한가?

  1. T만 쓰면 Java는 T가 숫자인지 문자인지 모릅니다.
  2. T extends Number라고 쓰면 T는 Integer, Double처럼 숫자 타입만 가능합니다.
  3. 그래서 doubleValue() 같은 Number의 메서드를 안전하게 호출할 수 있습니다.

09

제네릭 메서드

제네릭 메서드는 클래스 전체가 아니라 메서드 하나만 타입을 유연하게 받도록 만드는 문법입니다. 반환 타입 앞에 <T>를 적습니다.

class Printer {
    public static <T> void print(T value) {
        System.out.println(value);
    }
}

class Main {
    public static void main(String[] args) {
        Printer.print("Java");
        Printer.print(100);
        Printer.print(new Acorn("가람", 3, 80));
    }
}

Common Confusion

public static <T> void에서 <T>는 “이 메서드에서 T라는 타입 이름을 쓰겠다”는 선언입니다. 반환 타입이 T일 때도 있고, 위 예시처럼 void일 때도 있습니다.

10

와일드카드

와일드카드 ?는 “정확한 타입 이름은 모르지만 어떤 타입이 들어온다”는 뜻입니다. 메서드의 매개변수에서 여러 제네릭 타입을 받아야 할 때 자주 사용합니다.

import java.util.ArrayList;
import java.util.List;

class Main {
    static void printAll(List<?> list) {
        for (Object item : list) {
            System.out.println(item);
        }
    }

    static void printNumbers(List<? extends Number> list) {
        for (Number number : list) {
            System.out.println(number);
        }
    }

    static void addInteger(List<? super Integer> list) {
        list.add(10);
    }

    public static void main(String[] args) {
        ArrayList<String> names = new ArrayList<>();
        names.add("도토리");

        ArrayList<Integer> numbers = new ArrayList<>();
        numbers.add(10);

        printAll(names);
        printAll(numbers);
    }
}
문법 읽는 법 주요 용도
List<?> 어떤 타입이든 가능 읽기 중심 출력
List<? extends Number> Number 또는 Number의 자식 숫자를 꺼내서 읽을 때 사용
List<? super Integer> Integer 또는 Integer의 부모 Integer 값을 안전하게 넣을 때 사용

초보자 핵심 기억

extends 와일드카드는 주로 꺼내서 읽을 때, super 와일드카드는 주로 넣을 때 떠올리면 됩니다.

Answers

정답과 해설

  1. 정답: 클래스가 반드시 지켜야 하는 기능 약속입니다. 해설: 인터페이스는 메서드 이름과 사용 방식을 정하고, 실제 동작은 구현 클래스가 작성합니다.
  2. 정답: Printable 인터페이스를 구현한다는 뜻입니다. 해설: 따라서 Printable에 있는 추상 메서드를 클래스가 완성해야 합니다.
  3. 정답: Acorn 객체를 넣을 수 있습니다. 해설: 제네릭 타입이 Acorn으로 정해졌기 때문에 다른 타입을 넣으면 컴파일 오류가 납니다.
  4. 정답: compareTo입니다. 해설: Collections.sort(list)는 이 메서드를 사용해 객체끼리 순서를 비교합니다.
  5. 정답: this 객체가 앞에 옵니다. 해설: 비교 결과가 음수이면 첫 번째 객체를 더 작은 값으로 판단합니다.
  6. 정답: 정렬 기준을 여러 개 만들거나 클래스 밖에서 정렬 기준을 정하고 싶을 때 사용합니다. 해설: 이름순, 점수순처럼 기준을 바꿀 수 있습니다.
  7. 정답: 인터페이스 이름으로 호출합니다. 해설: 예를 들어 StudyTool.guide()처럼 씁니다.
  8. 정답: 나중에 정해질 타입 자리입니다. 해설: Box<String>이면 T가 String처럼 동작합니다.
  9. 정답: Number 또는 Number를 상속한 숫자 타입만 허용합니다. 해설: 그래서 Integer, Double 같은 숫자 타입을 받고 doubleValue() 같은 메서드를 안전하게 사용할 수 있습니다.
  10. 정답: 정확한 타입은 모르지만 어떤 타입이든 올 수 있다는 뜻입니다. 해설: 여러 타입의 리스트를 출력처럼 읽기 중심으로 처리할 때 편리합니다.
