서블릿 include 개념정리

include는 서블릿에서 사용하는 기능으로, 현재 응답 안에 다른 JSP나 서블릿의 실행 결과를 포함시키는 것이다.

쉽게 말하면 현재 페이지에 다른 페이지의 일부 내용을 가져와 붙이는 기능이다.

include의 의미

include는 요청을 완전히 넘기는 것이 아니라, 현재 응답을 유지하면서 다른 자원의 결과를 함께 출력한다.

예를 들어 현재 페이지가 메인 페이지라면, 그 안에 공통 메뉴나 하단 영역을 포함할 수 있다.

메인 페이지
 ├ 상단 메뉴 포함
 ├ 본문 내용 출력
 └ 하단 영역 포함
include의 역할

include는 공통으로 반복되는 화면을 재사용할 때 많이 사용한다.

예를 들어 여러 페이지에 같은 상단 메뉴가 들어간다면, 모든 JSP에 메뉴 코드를 직접 쓰는 대신 메뉴 부분을 따로 만들고 include로 불러올 수 있다.

이렇게 하면 같은 코드를 반복해서 쓰지 않아도 된다.

include의 흐름
사용자 요청
→ 현재 서블릿 또는 JSP 실행
→ 다른 JSP나 서블릿 결과를 include
→ 현재 응답에 포함
→ 하나의 화면으로 출력

즉, 현재 페이지가 중심이고 다른 결과를 중간에 끼워 넣는 구조이다.

include를 사용하는 이유

include를 사용하면 코드 중복을 줄일 수 있다.

상단 메뉴, 사이드바, 푸터처럼 여러 페이지에서 공통으로 쓰는 부분을 따로 관리할 수 있기 때문이다.

또한 공통 부분을 수정해야 할 때 한 파일만 수정하면 되기 때문에 유지보수가 쉬워진다.

forward와 include 차이

forward는 요청 처리를 다른 자원에게 완전히 넘기는 것이다.

반면 include는 현재 응답 안에 다른 자원의 결과를 포함하는 것이다.

forward = 너가 대신 처리해줘
include = 네 결과를 여기에 붙여줘
핵심 정리

서블릿 include는 현재 페이지를 만들면서 다른 JSP나 서블릿의 결과를 함께 포함하는 기능이다.

주로 공통 메뉴, 헤더, 푸터처럼 여러 페이지에서 반복되는 부분을 재사용할 때 사용한다.
