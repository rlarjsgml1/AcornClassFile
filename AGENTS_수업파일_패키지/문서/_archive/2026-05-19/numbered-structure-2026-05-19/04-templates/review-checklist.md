# Review Checklist

수정 후 확인할 항목입니다.

## 코드

- [ ] 컴파일 오류가 없는가?
- [ ] 파일명과 public class 이름이 맞는가?
- [ ] 패키지명과 폴더 경로가 맞는가?
- [ ] 기존 코드 스타일을 과하게 바꾸지 않았는가?
- [ ] 요청 범위를 벗어난 변경이 없는가?

## 문서

- [ ] 문서 목적이 첫 부분에 드러나는가?
- [ ] 실제 경로와 파일명을 기준으로 썼는가?
- [ ] 확인한 내용과 추정 내용을 구분했는가?
- [ ] 같은 이름의 문서가 불필요하게 반복되지 않는가?

## Git

- [ ] SourceTree staged files를 확인했는가?
- [ ] 제외해야 할 파일이 포함되지 않았는가?
- [ ] 커밋 메시지가 변경 목적을 설명하는가?
