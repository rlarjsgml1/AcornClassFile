# Commit Message Template

SourceTree 커밋 메시지 작성 기준입니다.

## 기본 형식

```text
영역 작업내용
```

## 예시

```text
Java2 Day3 실습 파일 추가
Java2 Day3 컴파일 오류 수정
WEB_CLOUD ajax 실습 파일 추가
AGENTS 문서 구조 실무형으로 정리
README 저장소 구조 설명 보완
```

## 기준

- 한 줄만 봐도 변경 목적이 보여야 합니다.
- 너무 긴 설명은 커밋 본문이나 개발로그에 적습니다.
- `수정`, `작업`, `테스트`처럼 의미가 약한 메시지는 피합니다.
