# Safety And Approval Policy

위험하거나 되돌리기 어려운 작업의 기준입니다.

## 사용자 확인이 필요한 작업

- 파일 삭제
- 대량 파일 이동 또는 대량 수정
- 패키지 설치 또는 삭제
- DB 데이터 변경
- 마이그레이션 실행
- 배포 작업
- Git push, merge, rebase, reset, force push
- 테스트 약화 또는 삭제
- 보안 설정 변경
- IDE, SDK, Gradle 버전 변경

## 보안

- 비밀번호, 토큰, API key, DB 접속 정보는 코드에 직접 넣지 않습니다.
- 민감 정보는 로그와 화면 출력에 남기지 않습니다.
- `.env`, `local.properties`, keystore, 인증 파일은 Git에 올리지 않습니다.

## 파일 정리 원칙

- 삭제보다 archive 보관을 우선합니다.
- archive에는 날짜를 붙입니다.
- archive로 옮긴 이유를 개발로그에 남깁니다.

## 실패 처리

- 에러를 숨기지 않습니다.
- 단순 경고를 치명적 오류처럼 과장하지 않습니다.
- 치명적 오류를 단순 경고처럼 축소하지 않습니다.
