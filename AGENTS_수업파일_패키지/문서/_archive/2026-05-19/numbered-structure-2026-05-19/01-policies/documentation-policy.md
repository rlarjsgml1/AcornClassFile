# Documentation Policy

README, 수업 정리, 사용방법 문서, 로그 문서 작성 기준입니다.

## 문서 작성 원칙

- 목적을 첫 부분에 씁니다.
- 실제 경로와 파일명을 확인한 뒤 씁니다.
- 확인한 내용과 추정한 내용을 분리합니다.
- 실행하지 않은 검증은 기록하지 않습니다.
- 한 문서에 모든 규칙을 몰아넣지 않습니다.

## 문서 위치 기준

- 패키지 구조 설명: `00-overview`
- 공통 규칙: `01-policies`
- 기술별 규칙: `02-engineering`
- 프로젝트별 규칙: `03-projects`
- 반복 양식: `04-templates`
- 실제 기록: `05-logs`

## 파일명 기준

- 파일명은 영어 소문자와 하이픈을 사용합니다.
- 같은 이름의 `README.md`를 여러 폴더에 만들지 않습니다.
- 문서의 역할이 드러나는 이름을 사용합니다.

예:

```text
java-eclipse-guide.md
debugging-log-template.md
git-sourcetree-policy.md
```
