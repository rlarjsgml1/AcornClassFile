# Data Model Guide

DTO, VO, Entity, DB 테이블, JSON 구조 정리 기준입니다.

## 작성 기준

- 필드명, 타입, 의미를 함께 적습니다.
- DB 컬럼과 Java 필드명이 다르면 매핑 관계를 적습니다.
- null 가능 여부와 기본값을 확인합니다.
- 화면 입력값, 서버 내부값, DB 저장값을 구분합니다.

## 예시 형식

```text
모델명:
사용 위치:
필드:
  - name: String, 사용자 이름
  - age: int, 나이
검증:
  - name은 비어 있으면 안 됨
DB 매핑:
  - name -> USER_NAME
```

## 주의

- 확인 전까지 테이블 구조를 임의로 만들지 않습니다.
- DTO에 화면 로직이나 DB 연결 코드를 넣지 않습니다.
- 비밀번호 같은 민감 값은 출력 로그에 남기지 않습니다.
