# UI UX Guide

화면 디자인, UI 구성, 사용자 흐름 기준입니다.

## 기본 기준

- 기존 화면의 색상, 여백, 폰트 흐름을 우선 따릅니다.
- 정보가 많은 화면은 표, 리스트, 섹션으로 정리합니다.
- 버튼, 입력창, 링크의 역할이 바로 보여야 합니다.
- 모바일과 데스크톱에서 텍스트가 겹치지 않아야 합니다.
- 장식보다 사용자가 해야 할 행동을 우선합니다.

## 화면 구성

- 제목은 화면 목적을 설명합니다.
- 주요 행동 버튼은 찾기 쉬워야 합니다.
- 입력 폼은 label, placeholder, validation 메시지를 구분합니다.
- 반복 항목은 같은 간격과 같은 정렬을 유지합니다.
- 오류가 났을 때 사용자가 다음 행동을 알 수 있어야 합니다.

## 웹 수업 화면 기준

- JSP 화면은 서버 데이터와 사용자 입력 데이터를 구분합니다.
- 폼 제출 후 성공, 실패, 빈 값 상태를 확인합니다.
- Ajax 화면은 요청 중, 성공, 실패 상태를 구분합니다.
