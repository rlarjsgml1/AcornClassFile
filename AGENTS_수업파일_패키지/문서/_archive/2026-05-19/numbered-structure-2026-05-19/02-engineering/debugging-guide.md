# Debugging Guide

에러 분석, 로그 해석, 디버깅 작업 기준입니다.

## 필수 정보

에러 질문에서 아래 정보가 없으면 원인을 단정하지 않습니다.

- 에러 메시지
- 에러 발생 위치
- 관련 코드 일부
- 기대한 동작
- 실제 발생한 동작
- 실행 환경 또는 사용 도구

## 분석 순서

1. 문제를 한 문장으로 요약합니다.
2. 확인된 사실을 먼저 정리합니다.
3. 가능성이 높은 원인부터 나눕니다.
4. 가장 작은 수정으로 먼저 해결합니다.
5. 수정 후 검증 방법을 제시합니다.
6. 같은 문제가 재발하지 않도록 예방 방법을 남깁니다.

## 언어별 확인

- Java: 컴파일 오류, 런타임 예외, 클래스패스
- Java Web: URL, servlet mapping, context path, 서버 로그
- Android: Gradle, Logcat, 생명주기, null 처리
- JavaScript: Console, Network, stack trace, 비동기 흐름
- DB: SQL 에러 코드, 쿼리, 바인딩 값, 트랜잭션
- Git: 브랜치, diff, 충돌 파일, 원격 상태
