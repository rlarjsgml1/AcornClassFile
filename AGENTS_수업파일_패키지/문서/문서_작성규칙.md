# 문서 작성 규칙

README, 수업 정리, 사용방법 문서, 로그 문서를 작성할 때 적용합니다.

## 작성 순서

1. 문서의 목적을 먼저 씁니다.
2. 적용되는 경로나 파일을 명확히 씁니다.
3. 사용자가 바로 따라 할 수 있는 순서로 정리합니다.
4. 확인한 내용과 추정한 내용을 분리합니다.
5. 실행하지 않은 검증은 실행했다고 쓰지 않습니다.

## 좋은 문서 기준

- 제목만 보고 용도를 알 수 있어야 합니다.
- 경로는 실제 저장소 기준으로 적습니다.
- 코드 설명은 화면 흐름이나 요청 흐름과 연결합니다.
- Git에 올릴 파일과 올리지 않을 파일을 구분합니다.
- 나중에 다시 봐도 왜 그렇게 했는지 알 수 있어야 합니다.

## 피해야 할 내용

- 확인하지 않은 실행 결과
- 존재하지 않는 파일명이나 패키지명
- 너무 긴 복붙 코드
- 수업 원본을 덮어쓰는 정리 방식
- 한 문서에 Git, 코드, 디자인, DB 규칙을 모두 섞는 방식
