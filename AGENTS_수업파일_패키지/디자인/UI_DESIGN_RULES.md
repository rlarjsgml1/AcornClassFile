# UI Design Rules

화면을 만들거나 수정할 때 적용합니다.

## 기본 기준

- 기존 화면의 색상, 여백, 폰트 흐름을 우선 따릅니다.
- 새 화면의 폰트, 여백, 박스 크기 기준은 `frontend/CSS_FOUNDATION_RULES.md`를 함께 확인합니다.
- 특정 브랜드 느낌을 참고할 때는 `REFERENCE_BRAND_TRANSFORM_RULES.md`를 함께 확인하고 원본을 그대로 복사하지 않습니다.
- 정보가 많은 화면은 카드 남발보다 표, 리스트, 섹션으로 정리합니다.
- 버튼, 입력창, 링크의 역할이 바로 보여야 합니다.
- 모바일과 데스크톱에서 텍스트가 겹치지 않아야 합니다.
- 장식보다 사용자가 해야 할 행동을 우선합니다.

## 화면 구성

- 제목은 화면 목적을 설명합니다.
- 주요 행동 버튼은 한 화면에서 찾기 쉬워야 합니다.
- 입력 폼은 label, placeholder, validation 메시지를 구분합니다.
- 반복 항목은 같은 간격과 같은 정렬을 유지합니다.
- 이미지가 필요하면 실제 내용과 관련 있는 이미지를 사용합니다.
