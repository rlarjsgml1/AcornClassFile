# Reference Brand Transform Rules

사용자가 특정 브랜드 느낌을 참고해 달라고 할 때 적용합니다.
목표는 원본 복제가 아니라, 원본에서 보이는 디자인 원리를 다른 콘셉트로 변형하는 것입니다.

## 1. 기본 원칙

- 실제 브랜드의 로고, 이미지, UI 코드, 고유 문구를 그대로 복사하지 않습니다.
- 참고 브랜드명은 방향을 잡기 위한 기준으로만 사용합니다.
- 최종 결과물은 새 브랜드명, 새 문구, 새 정보 구조를 가져야 합니다.
- 확인한 사실과 디자인 추정은 구분합니다.
- 최신 브랜드 스타일이 중요하면 공식 사이트, 뉴스룸, 디자인 공개 자료처럼 출처가 분명한 자료를 확인합니다.

## 2. 추출할 것과 피할 것

추출할 수 있는 것:

- 정보 밀도
- 네비게이션 구조
- 타이포그래피의 굵기와 크기감
- 색상 역할
- 버튼과 카드의 각진 정도
- 리스트, 랭킹, 칩, 탭 같은 정보 표현 방식
- 모션의 속도와 절제감

피해야 할 것:

- 공식 로고를 그대로 쓰기
- 공식 이미지, 상품 사진, 아이콘 무단 사용
- 공식 사이트의 HTML/CSS/JS 구조 복사
- 고유 슬로건이나 캠페인 문구 복사
- 원본 서비스명으로 사용자를 오해하게 만드는 화면 구성

## 3. 변형 절차

1. 참고 브랜드에서 보이는 시각 원리를 정리합니다.
2. 작업 대상의 실제 목적과 사용자 행동을 먼저 정합니다.
3. 새 브랜드명과 메뉴 체계를 만듭니다.
4. 색상과 폰트는 역할 토큰으로 바꿉니다.
5. 원본과 다른 문장, 다른 CTA, 다른 카드 정보를 작성합니다.
6. 모바일과 데스크톱에서 텍스트 겹침을 확인합니다.
7. 마지막 보고에서 "참고한 방향"과 "그대로 복제하지 않은 부분"을 구분합니다.

## 4. 패션 커머스형 참고 기준

무신사처럼 패션 커머스 느낌을 요청받았을 때는 아래 방향을 우선합니다.

```text
폰트        Pretendard 우선, Windows는 Malgun Gothic fallback
기본 색상    black / white / gray 중심
보조 색상    blue 계열 accent를 적게 사용
텍스트      12px, 14px, 16px 중심의 촘촘한 정보 계층
브랜드      두껍고 단단한 고딕 느낌
레이아웃    얇은 라인, 평면 카드, 모듈형 섹션
버튼        검정 primary, 흰색 outline 또는 subtle
메뉴        추천, 발매, 랭킹, 스냅, 브랜드/입점 같은 커머스 단어
모션        빠르고 짧은 hover, opacity, translate 중심
```

이 기준은 원본 사이트를 붙여 넣는 기준이 아닙니다.
예를 들어 `MUSINSA`가 아니라 새 콘셉트명인 `FRAME SEOUL`처럼 별도 브랜드로 변형합니다.

## 5. 색상 토큰 예시

```css
:root {
  --text: #000000;
  --text-soft: #333333;
  --muted: #666666;
  --subtle: #999999;
  --background: #ffffff;
  --surface: #ffffff;
  --surface-alt: #f5f5f5;
  --line: #ebebeb;
  --line-strong: #d8d8d8;
  --primary: #000000;
  --primary-hover: #222222;
  --accent: #2f6bff;
  --accent-soft: #edf3ff;
}
```

색상값은 프로젝트에 맞게 조정할 수 있지만, 역할명은 유지합니다.

## 6. 검토 질문

- 원본 브랜드로 착각할 만한 로고나 문구가 남아 있는가?
- 참고 브랜드의 느낌은 남고, 새 콘셉트의 이름과 정보 구조가 분명한가?
- 폰트, 크기, 색상, 버튼, 메뉴가 한 화면 안에서 같은 규칙을 따르는가?
- 모바일에서 긴 한국어 텍스트가 잘리지 않는가?
- 출처 확인이 필요한 최신 정보는 확인했는가?
