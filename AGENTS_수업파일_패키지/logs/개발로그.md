# 개발로그

저장소에서 진행한 개발, 정리, 문서화 작업을 기록합니다.

## 작성 규칙

- 날짜를 적습니다.
- 작업한 경로를 적습니다.
- 변경 이유를 적습니다.
- 검증 여부를 적습니다.
- 다음에 할 일을 짧게 남깁니다.

## 기록 템플릿

```text
## YYYY-MM-DD - 작업 제목

대상:
- path/to/file

변경:
- 무엇을 변경했는지

이유:
- 왜 변경했는지

검증:
- 실행한 검증
- 미검증이면 이유

다음:
- 이어서 확인할 일
```

## 2026-05-19 - AGENTS 문서 구조 분리

대상:
- AGENTS_수업파일_패키지/

변경:
- 문서, code, 규칙, 디자인, 모델, frontend, backend, logs 폴더를 추가했습니다.
- 기능별 README와 작업 기준 문서를 추가했습니다.
- 개발로그와 디버깅로그 파일을 추가했습니다.

이유:
- 에이전트 지침을 한 곳에 섞어두지 않고, 작업 종류별로 바로 찾을 수 있게 만들기 위해서입니다.

검증:
- 문서 파일 생성과 경로 목록을 확인했습니다.

다음:
- 실제 작업하면서 부족한 규칙을 각 폴더 문서에 추가합니다.
