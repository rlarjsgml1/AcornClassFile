# 디버깅로그

오류 원인, 수정 방법, 검증 결과를 기록합니다.

## 작성 규칙

- 에러 메시지를 그대로 적습니다.
- 발생 위치를 적습니다.
- 확인된 원인과 추정 원인을 구분합니다.
- 실제 수정한 내용을 적습니다.
- 검증 명령이나 확인 방법을 적습니다.

## 기록 템플릿

```text
## YYYY-MM-DD - 오류 제목

대상:
- path/to/file

증상:
- 에러 메시지 또는 화면 증상

확인:
- 실제로 확인한 사실

원인:
- 확인된 원인

수정:
- 수정한 내용

검증:
- 실행한 명령 또는 확인 방법

예방:
- 같은 문제가 다시 생기지 않게 확인할 점
```

## 2026-05-19 - Java2 Day3 실습학생 컴파일 오류

대상:
- AcornFileMain/Java2/workspace/java2prjDay3/src/day3/실습학생/day2Test_최정문.java

증상:
- `class day2Test is public, should be declared in a file named day2Test.java`

확인:
- 파일명은 `day2Test_최정문.java`였습니다.
- 내부 public class 이름은 `day2Test`였습니다.

원인:
- Java는 public class 이름과 파일명이 같아야 합니다.

수정:
- public class 이름을 `day2Test_최정문`으로 변경했습니다.

검증:
- `javac -encoding UTF-8` 컴파일 성공
- `java -cp ... day3.실습학생.day2Test_최정문` 실행 성공

예방:
- 새 Java 파일을 만들 때 파일명과 public class 이름을 먼저 맞춥니다.

## 2026-05-19 - 외부 UI 폴더 이름 변경 권한 오류

대상:
- `C:\VsHtml\UI파일모음집\Header파일\First`

증상:
- `Move-Item : 'FrameSeoul' 경로에 대한 액세스가 거부되었습니다.`
- `Move-Item : 경로의 일부를 찾을 수 없습니다.`

확인:
- 작업 대상은 현재 기본 쓰기 범위인 `C:\AcornFile` 밖의 `C:\VsHtml` 경로였습니다.
- 일반 실행으로는 `First` 폴더를 `FrameSeoul`로 이름 변경할 수 없었습니다.

원인:
- 외부 경로 쓰기 작업에는 권한 승인이 필요했습니다.

수정:
- 같은 부모 폴더 안에서 원본 경로와 대상 경로를 검증한 뒤 승인된 실행으로 `Move-Item`을 수행했습니다.

검증:
- `C:\VsHtml\UI파일모음집\Header파일\FrameSeoul` 폴더가 생성된 것을 확인했습니다.
- 새 폴더 안의 `index.html`, `css/style.css`, `html/*.html`, `js/script.js` 파일 목록을 확인했습니다.
- 기존 `First` 경로가 더 이상 존재하지 않는 것을 확인했습니다.

예방:
- 외부 경로 폴더명 변경은 `규칙/FOLDER_NAMING_RULES.md`의 경로 검증 패턴을 사용합니다.
- 쓰기 권한이 없는 경로에서는 우회하지 않고 승인을 요청합니다.
