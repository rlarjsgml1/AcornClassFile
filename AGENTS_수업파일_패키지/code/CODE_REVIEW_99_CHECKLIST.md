# Code Review 99 Checklist

코드 작업을 99점 수준으로 점검하기 위한 체크리스트입니다.
작업 성격에 맞는 항목을 적용하고, 적용하지 않는 항목은 이유를 남깁니다.

## 공통

- [ ] 요청 범위 안에서만 수정했는가?
- [ ] 기존 구조와 스타일을 따랐는가?
- [ ] 이름이 역할을 설명하는가?
- [ ] 불필요한 리팩터링이 없는가?
- [ ] 하드코딩이 필요한 경우인지 확인했는가?
- [ ] 보안값이 코드나 로그에 없는가?
- [ ] 새 의존성을 추가했다면 이유와 영향 범위를 기록했는가?
- [ ] 자동 생성 파일과 직접 작성한 파일을 구분했는가?

## Java

- [ ] package 선언과 폴더 경로가 맞는가?
- [ ] public class 이름과 파일명이 맞는가?
- [ ] import가 정리되어 있는가?
- [ ] null, 문자열 비교, 숫자 변환 문제를 확인했는가?
- [ ] 컴파일 또는 Eclipse Problems 확인을 했는가?

## Java Web

- [ ] URL, method, servlet mapping이 맞는가?
- [ ] JSP, Servlet, Service, DAO 역할이 분리되어 있는가?
- [ ] request/session attribute 이름이 일관되는가?
- [ ] 404, 500, DB 오류 가능성을 확인했는가?

## JavaScript

- [ ] DOM 요소가 존재하는지 확인했는가?
- [ ] 이벤트가 중복 바인딩되지 않는가?
- [ ] fetch/ajax 성공과 실패를 모두 처리했는가?
- [ ] Console과 Network를 확인했는가?

## SQL

- [ ] SQL injection 위험이 없는가?
- [ ] WHERE 없는 UPDATE/DELETE가 아닌가?
- [ ] transaction 기준이 필요한지 확인했는가?
- [ ] 변경 전후 데이터를 확인했는가?

## 검증

- [ ] 실행 가능한 최소 검증을 수행했는가?
- [ ] 실패한 검증을 숨기지 않았는가?
- [ ] 검증하지 못한 항목을 미확인으로 기록했는가?
- [ ] Git/SourceTree 변경 파일 목록을 확인했는가?
