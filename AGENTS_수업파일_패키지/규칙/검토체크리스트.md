# 검토 체크리스트

수정 후 마지막으로 확인할 항목입니다.

## 코드

- [ ] 컴파일 오류가 없는가?
- [ ] 파일명과 public class 이름이 맞는가?
- [ ] 패키지명과 폴더 경로가 맞는가?
- [ ] 기존 코드 스타일을 과하게 바꾸지 않았는가?
- [ ] 요청 범위를 벗어난 변경이 없는가?
- [ ] 정적 UI 작업에서 모든 관련 HTML 페이지의 헤더 문구가 일관되는가?
- [ ] 이전 콘셉트 단어가 남아 있지 않은가?

## 문서

- [ ] 문서 목적이 첫 부분에 드러나는가?
- [ ] 실제 경로와 파일명을 기준으로 썼는가?
- [ ] 확인한 내용과 추정 내용을 구분했는가?
- [ ] 오래된 문서 링크가 깨지지 않는가?
- [ ] 새로 추가한 문서가 `AGENTS.md` 또는 해당 폴더 가이드에서 연결되는가?

## 화면

- [ ] 데스크톱과 모바일에서 텍스트가 겹치지 않는가?
- [ ] 모바일 메뉴의 `aria-expanded` 상태가 맞는가?
- [ ] 특정 브랜드 참고 작업에서 원본 로고, 이미지, 코드, 고유 문구를 그대로 복사하지 않았는가?
- [ ] 결과물 폴더명이 실제 콘셉트와 맞는가?

## Git

- [ ] SourceTree staged files를 확인했는가?
- [ ] 제외해야 할 파일이 포함되지 않았는가?
- [ ] 커밋 메시지가 변경 목적을 설명하는가?
