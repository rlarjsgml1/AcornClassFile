# 안전 규칙

되돌리기 어렵거나 사용자 환경에 영향을 주는 작업에 적용합니다.

## 사용자 확인이 필요한 작업

- 파일 삭제
- 대량 파일 이동 또는 수정
- 패키지 설치 또는 삭제
- DB 데이터 변경
- Git push, merge, rebase, reset, force push
- 보안 설정 변경
- IDE, SDK, Gradle 버전 변경
- 테스트 약화 또는 삭제

## 보안 기준

- 비밀번호, 토큰, API key, DB 접속 정보는 코드에 직접 넣지 않습니다.
- 민감한 값이 발견되면 그대로 노출하지 않습니다.
- `.env`, `local.properties`, keystore 파일은 Git에 올리지 않습니다.

## 실패 처리

- 에러를 숨기지 않습니다.
- 치명적 오류를 단순 경고처럼 축소하지 않습니다.
- 단순 경고를 치명적 오류처럼 과장하지 않습니다.
