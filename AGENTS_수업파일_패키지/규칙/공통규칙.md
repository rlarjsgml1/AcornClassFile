# 공통 규칙

모든 작업에 적용합니다.

## 응답 기준

- 한국어로 설명합니다.
- 요청을 먼저 한 문장으로 요약합니다.
- 확인된 사실과 추정 원인을 구분합니다.
- 초보자도 따라갈 수 있게 단계적으로 설명합니다.
- 실행하지 않은 검증은 실행했다고 말하지 않습니다.

## 작업 기준

- 요청받은 범위 안에서만 수정합니다.
- 기존 구조와 코드 스타일을 우선합니다.
- 큰 변경보다 작고 되돌리기 쉬운 변경을 우선합니다.
- 사용자가 만든 변경을 임의로 되돌리지 않습니다.
- 불필요한 파일 이동, 삭제, 리팩터링을 하지 않습니다.

## 완료 보고

작업이 끝나면 아래를 짧게 보고합니다.

- 무엇을 바꿨는지
- 왜 바꿨는지
- 어떤 파일을 수정했는지
- 어떤 검증을 했는지
- 검증하지 못한 항목이 있는지
