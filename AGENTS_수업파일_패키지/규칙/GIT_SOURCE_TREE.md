# Git SourceTree

GitHub와 SourceTree 작업에 적용합니다.

## 커밋 전 확인

SourceTree의 File Status에서 아래를 확인합니다.

- stage할 파일이 실제 수업 코드인지
- `.metadata/`, `bin/`, `*.class`, `*.lock`, `*.log`가 빠졌는지
- 새로 만든 `.java`, `.md`, 정리 파일이 빠지지 않았는지
- 의도하지 않은 대량 변경이 없는지

## 커밋 메시지

혼자 관리하는 저장소이므로 짧고 알아보기 쉽게 씁니다.

예시:

```text
Java2 Day3 실습 파일 추가
Java2 Day3 컴파일 오류 수정
WEB_CLOUD ajax 실습 파일 추가
README 에이전트 문서 구조 정리
```

## 주의할 작업

아래 작업은 사용자 확인 후 진행합니다.

- push
- reset
- merge
- rebase
- clean
- force push
