# Backend Rules

서버 로직을 작성하거나 수정할 때 적용합니다.

## 1. 기본 원칙

- 백엔드는 요청, 업무 처리, 데이터 접근, 응답을 분리합니다.
- Servlet/JSP에 모든 로직을 몰아넣지 않습니다.
- Service는 업무 흐름과 트랜잭션 단위를 담당합니다.
- DAO는 SQL 실행과 DB 결과 변환을 담당합니다.
- DTO/VO는 데이터 전달만 담당합니다.
- JSP는 결과 출력과 최소한의 화면 분기만 담당합니다.

## 2. 요청 흐름

```text
요청 -> Controller/Servlet -> Service -> DAO/DB -> 응답
```

확인할 것:

- URL
- HTTP method
- context path
- servlet mapping
- request parameter 이름
- request body 형식
- response 형식

## 3. 입력 검증

- null, 빈 문자열, 숫자 변환, 날짜 형식, 길이 제한을 확인합니다.
- 사용자 입력값을 SQL 문자열에 직접 붙이지 않습니다.
- 화면 검증은 편의 기능이고, 서버 검증은 필수입니다.
- 파일 업로드는 확장자, 크기, 저장 경로, 파일명 충돌을 확인합니다.

## 4. 예외 처리

- 예외를 무시하거나 빈 catch로 숨기지 않습니다.
- 사용자에게 보여줄 메시지와 개발자가 볼 로그를 구분합니다.
- DB 예외는 SQL, 파라미터, 연결 상태를 확인할 수 있어야 합니다.
- 404, 400, 500은 원인과 확인 위치를 다르게 봅니다.

## 5. 로그 기준

- 로그에는 요청 경로, 주요 파라미터, 처리 결과, 예외 원인을 남깁니다.
- 비밀번호, 토큰, 주민번호, API key 같은 민감 정보는 로그에 남기지 않습니다.
- `System.out.println`은 수업 코드에서는 허용될 수 있지만, 실무형 기준에서는 logger 사용을 우선합니다.

## 6. 트랜잭션 기준

- 여러 INSERT/UPDATE/DELETE가 하나의 업무이면 같은 트랜잭션으로 묶습니다.
- 실패 시 rollback 기준을 정합니다.
- SELECT만 하는 조회와 데이터 변경 작업을 구분합니다.
- commit/rollback 위치는 Service 흐름 기준으로 판단합니다.

## 7. 응답 기준

- JSP 이동은 forward와 redirect 의도를 구분합니다.
- Ajax는 JSON 응답 형식을 명확히 합니다.
- 성공 응답과 실패 응답 구조를 일관되게 만듭니다.
- 목록 조회는 정렬, 검색, 페이징 기준을 확인합니다.

## 8. 성능 기준

- 반복문 안에서 DB를 여러 번 호출하지 않도록 확인합니다.
- 목록 조회는 필요한 컬럼만 조회합니다.
- 검색 조건과 정렬 컬럼은 인덱스 가능성을 고려합니다.
- 큰 파일, 큰 JSON, 큰 리스트는 페이지 단위로 처리합니다.

## 9. 환경 설정 기준

- DB URL, 계정, 비밀번호, API key는 코드에 직접 넣지 않습니다.
- 수업 프로젝트에서 코드에 들어간 설정값은 실무에서는 분리 대상임을 설명합니다.
- 로컬, 테스트, 운영 환경에서 달라지는 값은 설정 파일이나 환경 변수로 분리합니다.
- 라이브러리 jar를 추가하면 왜 필요한지와 어디에 두었는지 기록합니다.

## 10. 테스트 기준

- 성공 케이스만 확인하지 않고 실패 케이스도 확인합니다.
- 필수값 누락, 잘못된 숫자, 없는 id, 권한 없음, DB 오류 가능성을 확인합니다.
- 등록/수정/삭제는 DB 변경 전후를 확인합니다.
- Ajax는 Network 탭에서 요청과 응답을 확인합니다.
- 검증하지 못한 항목은 완료 보고에 미확인으로 남깁니다.

## 11. 릴리즈 전 점검

- 임시 로그와 테스트 출력이 남아 있지 않은지 확인합니다.
- 민감 정보가 커밋 대상에 없는지 확인합니다.
- `.metadata/`, `bin/`, `*.class`, `*.log`가 포함되지 않았는지 확인합니다.
- SourceTree에서 staged files만 다시 확인합니다.

## 12. 오류 확인

- 404: URL, mapping, context path, JSP 위치
- 400: 요청 파라미터, JSON 형식, 숫자/날짜 변환
- 401/403: 로그인, 세션, 권한
- 500: 예외 stack trace, null, SQL, DB 연결
- DB 오류: SQL, 컬럼명, 연결 정보, 바인딩 값
