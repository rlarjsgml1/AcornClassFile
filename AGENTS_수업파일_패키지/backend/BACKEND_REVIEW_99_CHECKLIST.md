# Backend Review 99 Checklist

백엔드 문서와 코드 작업을 99점 수준으로 점검하기 위한 체크리스트입니다.
모든 항목을 무조건 만족해야 하는 것은 아니지만, 해당 작업에 적용되는 항목은 빠짐없이 확인합니다.

## 요청 흐름

- [ ] URL, HTTP method, context path가 확인됐는가?
- [ ] Servlet/Controller mapping이 실제 요청과 맞는가?
- [ ] 요청 파라미터 이름이 화면 코드와 맞는가?
- [ ] forward와 redirect 선택 이유가 명확한가?

## 역할 분리

- [ ] Servlet/Controller는 요청과 응답 제어에 집중하는가?
- [ ] Service는 업무 흐름을 담당하는가?
- [ ] DAO는 DB 접근만 담당하는가?
- [ ] DTO/VO는 데이터 전달만 담당하는가?
- [ ] JSP에 복잡한 비즈니스 로직이 들어가지 않았는가?

## DB와 트랜잭션

- [ ] SQL 문자열 조합 대신 바인딩 방식을 사용했는가?
- [ ] connection, statement, result set 정리가 되는가?
- [ ] 여러 데이터 변경 작업의 transaction 기준이 있는가?
- [ ] SELECT, INSERT, UPDATE, DELETE 영향 범위를 확인했는가?
- [ ] 목록 조회에 검색, 정렬, 페이징 기준이 있는가?

## 보안

- [ ] 서버 입력값 검증이 있는가?
- [ ] 인증과 권한 확인 위치가 명확한가?
- [ ] 비밀번호, 토큰, API key가 코드나 로그에 노출되지 않는가?
- [ ] XSS, CSRF 가능성을 작업 성격에 맞게 검토했는가?

## 예외와 로그

- [ ] 예외를 숨기지 않는가?
- [ ] 사용자 메시지와 개발 로그를 구분하는가?
- [ ] 404, 400, 500 원인을 구분해 확인했는가?
- [ ] 실패 시 재현 가능한 정보가 남는가?

## 검증

- [ ] 컴파일 또는 서버 실행 확인을 했는가?
- [ ] 브라우저 URL 또는 API 요청을 실제로 확인했는가?
- [ ] DB 변경이 있으면 변경 전후 데이터를 확인했는가?
- [ ] 검증하지 못한 항목을 미확인으로 기록했는가?

## 운영 준비

- [ ] 임시 로그와 테스트 출력이 남아 있지 않은가?
- [ ] 민감 정보가 코드, 로그, 커밋 대상에 없는가?
- [ ] 환경별 설정 분리 필요 여부를 확인했는가?
- [ ] SourceTree staged files를 확인했는가?
