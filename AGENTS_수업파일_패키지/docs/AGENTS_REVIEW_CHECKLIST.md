# AGENTS Review Checklist

AGENTS.md와 세부 지침 문서를 점검할 때 사용하는 체크리스트입니다.

## 1. 루트 AGENTS.md 점검

- [ ] 모든 작업에 공통으로 필요한 내용만 들어 있는가?
- [ ] 특정 언어, 프레임워크, DB 세부 규칙이 과하게 들어 있지 않은가?
- [ ] 서로 충돌하는 지침이 없는가?
- [ ] 사용자 성향과 실제 사용 도구가 반영되어 있는가?
- [ ] 큰 변경의 영향 범위를 설명하게 되어 있는가?
- [ ] 검증하지 않은 내용을 검증했다고 말하지 못하게 막는가?
- [ ] 세부 문서로 이동하는 기준이 명확한가?

## 2. 세부 문서 점검

- [ ] Java/Eclipse, Java Web, Android, JavaScript, Git, 디버깅 문서가 역할별로 분리되어 있는가?
- [ ] 세부 문서가 루트 문서와 중복되지 않는가?
- [ ] 특정 프로젝트의 가짜 경로나 구조를 만들어내지 않는가?
- [ ] 확인 전까지 단정하지 말라는 규칙이 유지되는가?
- [ ] 검증 기준이 실제 작업 흐름과 연결되어 있는가?

## 3. 이 저장소 전용 점검

- [ ] `.metadata/`, `bin/`, `*.class`, `*.lock`, `*.log` 제외 기준이 들어 있는가?
- [ ] SourceTree 사용 기준이 들어 있는가?
- [ ] Java2 수업 파일과 WEB_CLOUD 작업 기준이 분리되어 있는가?
- [ ] Android Studio와 Kotlin 기준이 포함되어 있는가?
- [ ] 새 도구를 추가할 때 넣어야 할 항목이 정리되어 있는가?
