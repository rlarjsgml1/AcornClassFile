# Debugging Guide

에러 분석, 로그 해석, 디버깅 작업에 적용합니다.

## 1. 필수 정보

에러 질문에서 아래 정보가 없으면 정확한 원인을 단정하지 않습니다.

- 에러 메시지
- 에러 발생 위치
- 관련 코드 일부
- 기대한 동작
- 실제 발생한 동작
- 실행 환경 또는 사용 도구

## 2. 분석 순서

- 문제를 한 문장으로 요약합니다.
- 확인된 사실을 먼저 정리합니다.
- 추정 원인을 가능성이 높은 순서로 나눕니다.
- 추가 확인이 필요한 부분을 명확히 말합니다.
- 가장 작은 수정으로 먼저 해결합니다.
- 수정 후 검증 방법을 제시합니다.
- 같은 문제가 재발하지 않도록 예방 방법을 설명합니다.

## 3. 언어별 확인 기준

- Java는 컴파일 오류, 런타임 예외, 클래스패스, 서버 로그를 확인합니다.
- Kotlin/Android는 Gradle, Logcat, 생명주기, null 처리 문제를 확인합니다.
- JavaScript는 Console, Network, stack trace, 비동기 흐름을 확인합니다.
- DB는 SQL 에러 코드, 쿼리, 바인딩 값, 트랜잭션을 확인합니다.
- Git은 현재 브랜치, diff, 충돌 파일, 원격 상태를 확인합니다.

## 4. 답변 형식

에러 해결 답변은 가능하면 아래 순서를 따릅니다.

1. 문제 요약
2. 확인된 사실
3. 추정 원인
4. 해결 방법
5. 수정 코드
6. 코드 설명
7. 다음 할 일
