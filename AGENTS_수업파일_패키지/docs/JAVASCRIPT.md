# JavaScript Guide

JavaScript, 브라우저, Node.js, VSCode, 프론트엔드 작업에 적용합니다.

## 1. 기본 흐름

- 프론트엔드 설명은 화면 → 이벤트 → 로직 → 상태 변경 → 결과 표시 흐름으로 정리합니다.
- 서버 연동 설명은 요청 → 서버 → 응답 → 화면 반영 흐름으로 정리합니다.
- 프로젝트가 순수 JS, Node.js, React, Vue, Express 중 무엇인지 먼저 확인합니다.
- package manager는 lock 파일을 확인하기 전까지 단정하지 않습니다.

## 2. 구현 원칙

- 기존 코드 스타일과 모듈 방식을 우선 따릅니다.
- CommonJS와 ES Module을 임의로 섞지 않습니다.
- DOM 조작, 이벤트 바인딩, 상태 변경 위치를 구분합니다.
- API URL, 토큰, 비밀값은 코드에 직접 넣지 않습니다.
- 비동기 코드는 Promise, async/await, 콜백 중 기존 방식에 맞춥니다.
- 불필요한 라이브러리 추가보다 기존 기능으로 해결 가능한지 먼저 확인합니다.

## 3. 에러 분석

- 브라우저 오류는 Console, Network, Elements 탭 기준으로 확인합니다.
- Node.js 오류는 터미널 로그, stack trace, 실행 명령어, 환경변수를 기준으로 확인합니다.
- CORS 문제는 브라우저 정책, 서버 응답 헤더, 요청 URL, 인증 정보를 함께 봅니다.

## 4. 검증 기준

- 브라우저 콘솔 오류 여부를 확인합니다.
- 실제 화면 동작을 확인합니다.
- API 연동은 Network 탭의 요청, 상태 코드, 응답을 확인합니다.
- 검증하지 못한 항목은 미확인으로 보고합니다.
